# Projeto Integrado II – Protótipo do MVP

## Descrição do Projeto

Este repositório contém os materiais desenvolvidos para o **Entregável Parcial 3 (EP3)** da disciplina **Projeto Integrado II**, do curso de **Análise e Desenvolvimento de Sistemas – UFCA**.

O objetivo desta etapa do projeto é desenvolver a **prototipação do MVP (Minimum Viable Product)** por meio da criação de **wireframes** e da definição do **sitemap do sistema**, permitindo visualizar a estrutura e o funcionamento da aplicação antes da implementação do software.

A prototipação permite compreender a organização das funcionalidades, a navegação entre as telas e a forma como os usuários irão interagir com o sistema.

---

# Objetivos do Entregável

* Desenvolver wireframes representando as telas principais do sistema.
* Criar o sitemap da aplicação.
* Organizar a estrutura de navegação do sistema.
* Documentar o processo de prototipação.
* Aplicar princípios de design centrado no usuário.

---

# Prototipação do Sistema

A prototipação foi realizada utilizando a ferramenta **Whimsical**, que permite criar wireframes e representar visualmente as interfaces do sistema.

Foram desenvolvidas interfaces para diferentes perfis de usuários do sistema, incluindo:

* Administrador
* Professor
* Responsável

Esses protótipos simulam o funcionamento da plataforma e demonstram como os usuários poderão acessar e utilizar as funcionalidades do sistema.

---

# Wireframes

Os wireframes representam a estrutura visual das principais telas do sistema. Eles foram criados com o objetivo de demonstrar como as funcionalidades estarão organizadas dentro da interface da aplicação.

Entre as telas desenvolvidas estão:

* Dashboard do sistema
* Painel do administrador
* Painel do professor
* Painel do responsável

Durante a criação dos wireframes foram considerados princípios importantes de usabilidade, como:

* organização clara das informações
* facilidade de navegação
* interface simples e intuitiva
* boa experiência do usuário

---

# Sitemap do Sistema

O sitemap apresenta a estrutura de navegação da aplicação, demonstrando como as páginas estão organizadas dentro do sistema.

Estrutura do sistema:

Sistema
├── Login
├── Dashboard

Administrador
├── Gerenciar usuários
├── Gerenciar alunos

Professor
├── Registrar atividades
├── Visualizar alunos

Responsável
├── Visualizar desempenho do aluno
├── Acompanhar atividades

O sitemap facilita a compreensão da arquitetura do sistema e auxilia no planejamento das etapas futuras de desenvolvimento.

---

# Design Centrado no Usuário

Durante o desenvolvimento dos wireframes foram aplicados princípios de **Design Centrado no Usuário (User-Centered Design)**.

Essa abordagem busca desenvolver sistemas considerando as necessidades, expectativas e limitações dos usuários finais.

Quando um sistema é desenvolvido com foco no usuário, ele tende a ser:

* mais intuitivo
* mais fácil de utilizar
* mais eficiente
* mais acessível

A área de **Interface Humano-Computador (IHC)** tem grande importância nesse processo, pois contribui para a criação de interfaces mais amigáveis e eficientes.
